# Repo Instructions
- Refer to Dropbox Sign API documentation found here https://developers.hellosign.com/api
- Use `$dropboxsign-mcp-setup` when configuring this repo for first-time local use, environment variables, branding assets, MCP client registration, or first-run verification.
- When troubleshooting contract styling, treat `dropboxsign_auth_status.templatesDir` as the source of truth for where template files live. For stable custom branding across client launch locations, recommend `DROPBOXSIGN_TEMPLATE_DIR` as an absolute path containing at least `default.html` and `default.css`.
